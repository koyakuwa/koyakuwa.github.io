1 理論

文献Richter2016, Burri2015, Mellinger2011 を基に理論式を整理 する．

1.1 線形最適軌道

N-1 次元の多項式を以下のように示す．

\begin{align} p(t) &= \boldsymbol{c} \cdot \boldsymbol{t}\\ \boldsymbol{t} &= [1 ~t ~t^2 ~\cdots ~t^{N-1}]\\ \boldsymbol{c} &= [c_0 ~\cdots ~c_{N-1}]^{\mathrm{T}} \end{align}

今，軌道がD 個のM次元の多項式で構成されているとすると， 全体の評価式は，

\begin{equation} \label{eq-j_pol1} J_{polynomials} = \sum_{i =1}^M \sum_{d =1}^D \int_0^{T_{s,i}} \sum_{j=0}^{N-1} \left \| \frac{d^j p_{i,d}(t)}{dt^j}\right \| \cdot w_j \end{equation}

このとき，\(i\) 個目の多項式の評価関数は，

\begin{align} J_{i} &= \int_0^T w_0 ~p(t)^2 + w_1 ~p'(t)^2\\ \nonumber & + w_2 ~p''(t)^2 + \cdots + w_N ~p^{(N)}(t)^2\\ \nonumber &= \boldsymbol{c}_{i}^{\mathrm{T}} ~Q(T) ~\boldsymbol{c}_{i} \end{align}

ここでスナップ（加速度の微分の微分）を最小化する軌道を考えるので， \(w_4 =1\) として，その他の\(w_j\) はすべてゼロに設定する．

このとき，式(\ref{eq-j_pol1})は

\begin{align} &J_{polynomials} =\\ \nonumber &\begin{bmatrix} \boldsymbol{c}_{1,4}\\ \vdots\\ \boldsymbol{c}_{M,4} \end{bmatrix} \begin{bmatrix} Q_{1}(T_1) & & \\ & \ddots&\\ & & Q_M(T_M) \end{bmatrix} \begin{bmatrix} \boldsymbol{c}_{1,4}\\ \vdots\\ \boldsymbol{c}_{M,4} \end{bmatrix}^{\mathrm{T}} \end{align}

次に，任意の点を多項式軌道が通過する制約条件として

\begin{equation} A_i c_i = d_i, ~ A_i = \begin{bmatrix} A_0\\ A_T \end{bmatrix}, ~ d_i = \begin{bmatrix} d_0\\ d_T \end{bmatrix} \end{equation}

が導入される．

Symbol Defenition
\(A\) 多項式の変数と多項式の導関数の間の遷移行列
\(d_i\) \(i_{th}\) 番目の線分の最初（\(d_0\) ） 最後（\(d_T\) ）の微分値を含む

1.2 非線形最適軌道

到達時間を評価式に導入する場合は

\begin{equation} \label{eq-j_pol2} J = J_{polynomials} + k_{T} \cdot \left( \sum_{i=1}^{M} T_{s,i}\right)^2 \end{equation}

2.2 bool PolynomialOptimization<_N>::solveLinear()

template <int _N>
bool PolynomialOptimization<_N>::solveLinear() {
  CHECK(derivative_to_optimize_ >= 0 &&
        derivative_to_optimize_ <= kHighestDerivativeToOptimize);
  // Catch the fully constrained case:
  if (n_free_constraints_ == 0) {
    LOG(WARNING)
        << "No free constraints set in the vertices. Polynomial can "
           "not be optimized. Outputting fully constrained polynomial.";
    updateSegmentsFromCompactConstraints();
    return true;
  }

  // TODO(acmarkus): figure out if sparse becomes less efficient for small
  // problems, and switch back to dense in case.

  // Compute cost matrix for the unconstrained optimization problem.
  // Block-wise H = A^{-T}QA^{-1} according to [1]
  Eigen::SparseMatrix<double> R;
  constructR(&R);

  // Extract block matrices and prepare solver.
  Eigen::SparseMatrix<double> Rpf = R.block(
      n_fixed_constraints_, 0, n_free_constraints_, n_fixed_constraints_);
  Eigen::SparseMatrix<double> Rpp =
      R.block(n_fixed_constraints_, n_fixed_constraints_, n_free_constraints_,
              n_free_constraints_);
  Eigen::SparseQR<Eigen::SparseMatrix<double>, Eigen::COLAMDOrdering<int>>
      solver;
      solver.compute(Rpp);

  // Compute dp_opt for every dimension.
  for (size_t dimension_idx = 0; dimension_idx < dimension_; ++dimension_idx) {
    Eigen::VectorXd df =
        -Rpf * fixed_constraints_compact_[dimension_idx];  // Rpf = Rfp^T
    free_constraints_compact_[dimension_idx] =
        solver.solve(df);  // dp = -Rpp^-1 * Rpf * df, 線形方程式を逆行列を求めずに解く.
  }

  updateSegmentsFromCompactConstraints();
  return true;
}

2.3.1 QR 分解

wikipedia より

同様に，QR分解は A を直交行列 Q と上三角行列 R の積 QRとして表す． 系 Q(Rx) = b は Rx = tQb = c によって解かれ， 系 Rx = cは後退代入によって解かれる． 必要な加法と乗法の回数はLU分解のときの約2倍だが， QR分解は数値的に安定なため不正確な計算においてより多くの桁数が必要とならない．
